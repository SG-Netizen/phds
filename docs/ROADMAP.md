## 当前版本：v0.2.0（技术原型）

### 已完成交付

- Ed25519 + SM2 双密钥体系
- AES-256-GCM 认证加密 + ECIES 密钥交换
- 手机号+PIN 简化密钥派生（PBKDF2）
- JWT 授权、撤销、日志全链路
- 轻量 PoW 本地链存证
- FastAPI 接口服务（6 个接口）
- 5 套独立 Demo + 40 项单元测试
- 完整架构文档与部署指南
- 自定义 BioRecord 数据结构

## 短期（v0.3.0）— 安全性增强与医疗标准对接

**目标**：补齐安全体系短板、建立外部信任、打通 HIS 对接门槛

> 全部为新增功能与扩展接口，不破坏 v0.2.0 核心协议兼容性。原有手机号+PIN 简化方案保留，新能力均为可选扩展。

### 1. 密钥恢复机制（SSS 密钥分片）

**边界说明**：可选扩展，默认保留原有「手机号+PIN」派生路径，不破坏现有协议兼容性。用户主动启用后才生成分片。

- [ ] 实现 Shamir Secret Sharing（GF(256) 或调用 `ssspy`）
- [ ] 用户可自选阈值 N/M，生成多份分片
- [ ] 分片分发接口（导出为文本/二维码）
- [ ] 达到阈值后恢复根密钥
- [ ] 默认关闭，不影响现有简化流程
- [ ] 配套 `demo/demo_recovery.py` 演示脚本

### 2. 密钥轮换接口

**边界说明**：患者主动发起轮换，旧密钥设置过渡期后平滑失效，旧授权凭证在此期间可降级解密。

- [ ] `keys.py` 增加 `rotate_keypair()` 方法
- [ ] 生成新密钥对，对已加密病历重新加密
- [ ] 旧密钥过渡期支持降级解密（默认 30 天）
- [ ] 过渡期满后旧授权凭证自动失效
- [ ] 轮换事件写入存证日志

### 3. FHIR 适配器

**边界说明**：转换层实现，不重构底层 BioRecord 内核，最小侵入改造。BioRecord 仍为内部协议数据模型，FHIR 仅做外部互操作格式。

- [ ] 引入 `fhir.resources` 库
- [ ] 实现 `FHIRAdapter` 类：`BioRecord <-> FHIR Observation/Patient/Bundle`
- [ ] 双向序列化（导出 / 导入 FHIR JSON）
- [ ] 扩展 API 接口支持 FHIR 格式请求/响应
- [ ] 配套 `demo/demo_fhir.py` 演示脚本
- [ ] `docs/FHIR_ADAPTER.md` 使用文档

### 4. 公开安全自评文档

**边界说明**：自评 ≠ 第三方密码安全审计，仅用于透明展示协议设计思路与技术选型依据，不具备商用审计效力。

- [ ] `docs/SECURITY_SELF_ASSESSMENT.md`
- [ ] 密码原语选型清单与参数依据
- [ ] 密钥全生命周期攻击面分析（生成 → 存储 → 传输 → 销毁）
- [ ] 已知限制清单（内存劫持、侧信道、Python 运行时风险）
- [ ] 安全部署建议（HSM/SE 硬件、生产环境禁止真实数据）
